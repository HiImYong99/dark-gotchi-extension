# 🚀 Dark-gotchi 배포 및 가이드 (Deployment Guide)

이 문서는 개발 완료된 **Dark-gotchi** 확장 프로그램을 크롬 웹 스토어에 배포하거나 로컬 생산 환경에서 테스트하는 방법을 설명합니다.

---

## 1. 빌드 및 패키징 (Build & Packaging)

배포 전, 모든 소스 코드를 최적화(Minify)하고 하나의 압축 파일로 만드는 과정입니다.

1.  **빌드 실행**: 터미널에서 프로젝트 루트 디렉토리로 이동 후 아래 명령어를 실행합니다.
    ```bash
    node scripts/build.cjs
    ```
2.  **결과물 확인**:
    *   `dist/`: 최적화된 소스 코드가 포함된 폴더입니다.
    *   `dist.zip`: 웹 스토어 업로드용 압축 파일입니다. (내부에 `manifest.json`이 루트에 위치해야 합니다.)

---

## 2. 로컬 최종 테스트 (Local Production Test)

업로드 전, 실제 배포될 파일들이 정상 작동하는지 확인합니다.

1.  크롬 브라우저에서 `chrome://extensions/` 주소로 접속합니다.
2.  우측 상단의 **개발자 모드(Developer mode)**를 활성화합니다.
3.  **압축해제된 확장 프로그램을 로드합니다(Load unpacked)** 버튼을 클릭합니다.
4.  프로젝트 루트의 `dist` 폴더를 선택합니다.
5.  아이콘이 올바르게 표시되는지, 팝업 및 캐릭터 기능이 정상인지 최종 확인합니다.

---

## 3. 크롬 웹 스토어 등록 (Chrome Web Store Submission)

### A. 개발자 계정 준비
*   [Chrome Developer Dashboard](https://chrome.google.com/webstore/devconsole/)에 접속합니다.
*   구글 계정으로 로그인 후, 최초 1회 개발자 등록비($5)를 결제해야 합니다.

### B. 새 항목 추가
1.  **[+ 새 항목]** 버튼을 클릭합니다.
2.  빌드된 **`dist.zip`** 파일을 업로드합니다.
3.  업로드가 완료되면 항목 수정 페이지로 이동합니다.

### C. 대시보드 정보 입력
*   **스토어 등록정보 (Store Listing)**:
    *   **설명**: 사용자가 읽을 앱 소개글을 작성합니다. (한국어/영어 모두 입력 권장)
    *   **아이콘**: `assets/icons/icon128.png` (128x128) 파일을 사용합니다.
    *   **스크린샷**: 확장 프로그램이 작동하는 모습(웹페이지의 펫, 팝업 UI 등)을 캡처하여 업로드합니다. (1280x800 또는 640x400 권장)
*   **개인정보 보호 (Privacy)**:
    *   **권한 요구 사유**: `tabs`, `storage` 등의 권한이 왜 필요한지 간단히 설명합니다. (예: "To save pet state and display it on active tabs")
    *   **데이터 사용**: 사용자 데이터를 수집하지 않는다면 해당 없음을 체크합니다.

---

## 4. 심사 및 출시 (Review & Publish)

1.  모든 정보 입력을 마치면 **[심사팀으로 제출]**을 클릭합니다.
2.  **심사 기간**: 보통 1~3일 정도 소요되지만, 권한이 많을 경우 최대 3주까지 걸릴 수 있습니다.
3.  **출시**: 심사가 통과되면 자동으로 스토어에 게시되거나, 수동 게시 설정을 했다면 직접 게시 버튼을 눌러야 합니다.

---

## 5. 업데이트 방법 (Updating)

새로운 기능을 추가하거나 버그를 수정한 경우:

1.  `dark-gotchi/manifest.json`의 `"version"` 번호를 올립니다 (예: `1.0.0` -> `1.0.1`).
2.  다시 `node scripts/build.cjs`를 실행하여 새 `dist.zip`을 만듭니다.
3.  개발자 대시보드에서 해당 항목으로 들어가 **[패키지 업로드]**를 클릭하여 새 파일을 올리고 다시 심사를 요청합니다.

---

### 💡 주의사항
*   `dist.zip` 내부의 `manifest.json` 파일이 루트 레벨에 있어야 합니다. (현재 빌드 스크립트는 이를 자동으로 보장합니다.)
